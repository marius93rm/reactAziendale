# Project work

Il progetto accompagna tutti i moduli. I partecipanti costruiscono una mini applicazione aziendale con dashboard, gestione dati, form, ruoli, performance e test.

## Cartelle

- [`starter/`](starter/) contiene la base consegnata ai partecipanti.
- [`soluzione/`](soluzione/) contiene l'implementazione di riferimento.
- [`documentazione/`](documentazione/) contiene brief, decisioni tecniche e checklist.

## Requisiti funzionali

- Routing tra dashboard, dettaglio e modifica dei record.
- Componenti condivisi per azioni, campi e messaggi di stato.
- Servizio dati con loading, error, retry ed empty state.
- Form multi-step con validazione e salvataggio controllato.
- Ruoli semplificati per mostrare o disabilitare azioni.
- Test sui flussi principali.

## Criteri di qualità

- Confini chiari tra feature e codice condiviso.
- Stato collocato nel livello corretto.
- Feedback accessibili per tastiera e screen reader.
- Ottimizzazioni motivate da misure.
- Test basati sul comportamento.
